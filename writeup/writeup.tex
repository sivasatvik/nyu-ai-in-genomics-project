\begin{abstract}
This project tests whether environmental measurements from biological sampling
locations can condition a protein sequence generator. We link GOLD organism
metadata to reviewed UniProt proteins, enrich sample coordinates with NASA POWER
weather and WorldClim climate variables, clean corrupted rows, and train an LSTM
to generate amino-acid sequences from temperature, precipitation, and
solar-radiation inputs. The cleaned dataset contains 8,631 protein rows,
including 2,716 rows from extreme Earth environments. The LSTM outperforms a
simpler RNN baseline on validation next-token prediction, and generated
sequences show weak environment-dependent motifs; InterProScan annotates five of
six generated sequences as type I antifreeze-like, likely driven by
alanine-rich low-complexity patterns. We also compare Earth training conditions
with exoplanet temperature and radiation distributions to show where model
prompts become extrapolations. Overall, this is a reproducible proof of concept,
not evidence that generated sequences are functional proteins.
\end{abstract}

\section{Background}
Proteins are chains of amino acids that fold into molecular machines. Their
sequences are shaped by ancestry, function, and physical environment. For
example, proteins from organisms living in cold, dry, hot, or high-radiation
habitats may show different sequence patterns because those environments place
different stresses on protein stability and cellular survival.

Our project combines four public resources. GOLD provides organism metadata,
including taxonomy IDs and geographic coordinates when available \cite{gold}.
UniProt provides reviewed protein sequences for those taxonomy IDs
\cite{uniprot}. NASA POWER provides daily surface temperature and radiation
\cite{nasa}. WorldClim provides long-term monthly climate rasters for
precipitation, solar radiation, average temperature, and vapour pressure
\cite{worldclim}. We also implemented SoilGrids pH and soil-carbon retrieval,
but those variables were not used in the final model because coverage and units
still need validation \cite{soilgrids}.

\section{Motivation}
Most protein generators are conditioned on sequence context, structure, or text
prompts. We explored a different question: can a model use the physical
environment associated with an organism to generate protein-like amino-acid
sequences? This matters because Earth contains organisms from extreme habitats,
but exoplanets can have much broader temperature and radiation regimes. Our
hypothesis was that a conditional sequence model would learn some environmental
signal from Earth data, while an exoplanet comparison would reveal where the
model is extrapolating beyond its training distribution.

\section{Data}
\subsection{Source Data}
The pipeline starts with GOLD organism records. The local cached GOLD table has
530,160 rows, of which 64,665 have both latitude and longitude. For each
coordinate-bearing taxonomy ID, the code queries the UniProt REST API for up to
five reviewed protein entries and retrieves accession, entry name, organism
name, and sequence. These protein rows are joined back to GOLD by NCBI taxonomy
ID.

Each unique location/date is then enriched with NASA POWER daily temperature
(\texttt{NASA\_Temp\_C}) and radiation (\texttt{NASA\_Radiation}). If a GOLD
collection date is missing or unparseable, the code uses 2015-01-01 as a
fallback. WorldClim rasters add monthly precipitation, solar radiation, average
temperature, and vapour pressure. The cleaning script removes rows containing
common no-data values such as \texttt{-9999}, \texttt{-32768}, \texttt{32767},
\texttt{65535}, or values larger than \texttt{1e30}.

\section{Methods}
\subsection{Inputs and Encoding}
We use three environmental input features: NASA daily temperature, WorldClim
total annual precipitation, and WorldClim mean annual solar radiation. These
features were chosen because they are available for most samples and represent
thermal stress, water availability, and radiation exposure. We standardize them
with a fitted \texttt{StandardScaler}.

The output is a protein sequence represented one amino acid at a time. The
vocabulary has 23 tokens: the 20 standard amino acids plus \texttt{<PAD>},
\texttt{<SOS>}, and \texttt{<EOS>}. Unknown amino-acid characters are currently
mapped to \texttt{<PAD>}, which is a limitation because rare residues are not
modeled explicitly.

\subsection{Models and Training}
The main model is a conditional LSTM generator. The three environmental inputs
are transformed into the LSTM's initial hidden and cell states. The model then
reads previous amino-acid tokens and predicts the next token at each position.
It uses 64-dimensional token embeddings, a hidden size of 256, batch size 32,
Adam with learning rate 0.001, and cross-entropy loss that ignores padding.

For comparison, we trained a simpler conditional RNN baseline with the same
inputs, vocabulary, embedding size, hidden size, optimizer, and row split. Both
models use a 90\% training and 10\% validation split with
\texttt{random\_state=42}. This prevents exact row overlap, but it is not a
strict biological holdout because related organisms or similar protein families
may appear in both sets.

\subsection{Evaluation}
We report validation cross-entropy loss, perplexity, and next-token accuracy
excluding padding. Perplexity measures how many plausible next tokens the model
is effectively choosing among; lower is better. We also generate one hot and one
cold example sequence and compute GRAVY, the grand average of hydropathy.
Positive GRAVY values are more hydrophobic; negative values are more
hydrophilic. We additionally generated six 100-amino-acid sequences across
temperate, cold, hot, desert, and ice-moon settings and submitted them to
InterProScan for protein-family annotation \cite{interpro}. Finally, we compare
Earth training environments against confirmed exoplanets from the NASA
Exoplanet Archive using temperature and insolation \cite{nasaexo}.

\section{Results}
\subsection{Data Distribution}
The cleaned dataset contains a substantial extreme-Earth subset. Using thresholds
of annual average temperature below $-10^\circ$C or above $50^\circ$C, annual precipitation below
50 mm or above 4,000 mm, or mean solar radiation above 18,000 kJ/m$^2$/day, we
found 2,716 extreme rows out of 8,631 cleaned rows (31.47\%). The data is a little bit skewed towards the cold extremes. This gives the model some exposure to challenging
Earth environments, but not necessarily to exoplanet-like conditions. The full
script output is included in Appendix~\ref{app:extreme}. We plotted the sequence space using PCA and t-SNE on 3-mer TF-IDF features to explore whether environmental variables align with sequence patterns (Figure~\ref{fig:sequence-space}).

\subsection{Model Performance}
The LSTM outperformed the simpler RNN baseline on the validation split
(Table~\ref{tab:baseline}). It had lower loss (seen in Figure~\ref{fig:loss-curves}) and perplexity and higher
next-token accuracy. The generated-sequence GRAVY results were mixed: the LSTM
produced a near-neutral hot sequence and a very hydrophilic cold sequence,
whereas the RNN baseline produced a one-residue hot sequence and a hydrophobic
cold sequence. The one-residue RNN output is a warning sign that the baseline
can terminate generation too early under extrapolated inputs.

\subsection{Generated Sequence Annotation}
To inspect whether generated sequences resembled known protein families, we
generated six sequences from different environmental prompts and submitted them
to InterProScan (full details in Appendix~\ref{app:generated}). All six outputs
started with methionine and reached the 100-amino-acid generation limit. Five
of six matched type I antifreeze protein (IPR000104), including cold, hot,
temperate-preset, desert, and ice-moon prompts. This is biologically plausible
for the cold cases because real type I antifreeze proteins are alanine-rich
alpha-helical proteins; however, the repeated antifreeze calls across several
unrelated prompts also reveal a limitation: the model often generates
low-complexity alanine/leucine-rich sequences. The clearest environmental signal
appears in the first 4--8 residues, where temperate prompts produce a
lysine/leucine motif, extreme cold expands the lysine/leucine motif, and the
85$^\circ$C desert prompt produces a threonine/leucine motif.

\subsection{Earth-to-Exoplanet Shift}
The exoplanet comparison directly addresses distribution shift. The model is
trained on Earth sampling sites, but confirmed exoplanets with equilibrium
temperatures between 200 K and 400 K can still have temperature and radiation
combinations outside the Earth training cloud. Figure~\ref{fig:exo} shows this
gap: Earth data provide useful extreme examples, but exoplanet-like prompts can
ask the model to extrapolate beyond the distribution it learned from.

\section{Discussion and Conclusion}
The project succeeds as an end-to-end genomic-AI pipeline: it links biological
records to protein sequences, enriches them with environmental variables,
trains conditional sequence models, evaluates a baseline comparison, and checks
how far exoplanet prompts shift away from Earth training data. The LSTM is
better than the RNN baseline on next-token prediction, but 25.42\% accuracy is
still modest and does not establish biological function.

The extreme-environment subset is also skewed toward cold conditions, so the
model sees many more Antarctic-like examples than hot or humid ones. A larger
and better balanced set of conditional examples across all regimes could reduce
that skew and likely improve how well the model responds to non-cold prompts.

The largest scientific limitation is distribution shift. Earth extremophiles
cover many difficult habitats, but exoplanets can differ in atmospheric
composition, pressure, liquid-water availability, gravity, orbital distance,
host-star type, stellar flare activity, ultraviolet spectrum, magnetic shielding,
and radiation chemistry. These variables, not just chemical concentrations, can
affect whether a sequence is plausible. Our current model only uses
temperature, precipitation, and solar radiation, so it cannot know whether an
exoplanet has a protective atmosphere, a nearby active star, or surface
conditions compatible with biochemistry.

Other limitations are biological. Proteins are shaped by function and ancestry,
not environment alone; random row splits can leak related proteins across train
and validation; nonstandard amino acids are not modeled; and generated sequences
are evaluated only with next-token metrics and simple hydropathy. Future work
should split by taxonomy or protein family, add an unconditional baseline, add
more environmental variables where reliable, and validate generated proteins
with folding, stability, and functional assays before making design claims.

\section{Reproducibility and Supplementary Materials}
All experiments were run from the project root. These exact commands reproduce
the main pipeline (assumes a Python virtual environment is activated at
`.venv'). Replace paths if you use different filenames or locations. Since 
the data and checkpoints are included in the repo, we can directly run from step 5.\\
1) Create merged UniProt + GOLD dataset (writes a Parquet at data/combined... this parquet is also included in the repo for convenience, but you can recreate it with this command). We might need to tweak the MAX\_TAX\_IDS and PROTEINS\_PER\_TAX\_ID constants in `src/raw\_data.py' as this is usually very time consuming:
\begin{verbatim}
source .venv/bin/activate
python src/raw_data.py
\end{verbatim}
2) Convert and clean the merged dataset (writes CSVs under `data/`):
\begin{verbatim}
python src/clean_data.py \
  --input-parquet data/combined_uniprot_and_gold_environmental_data_10K.parquet \
  --output-csv data/combined_uniprot_and_gold_environmental_data_10K.csv \
  --cleaned-output-csv data/cleaned_environmental_data_10K.csv
\end{verbatim}
3) Train the conditional LSTM. The checkpoints are already included in the repo, but you can retrain with this command:
\begin{verbatim}
python src/train.py \
  --data data/cleaned_environmental_data_10K.csv \
  --scaler checkpoints/environmental_scaler_10K.pkl \
  --checkpoint checkpoints/best_alien_protein_model_10K.pt \
  --loss-plot figures/train_loss_curve_lstm.png
\end{verbatim}
4) Train the baseline RNN for comparison (produces its own checkpoint and loss plot). The checkpoint for the baseline is also included, but you can retrain with this command:
\begin{verbatim}
python src/train_baseline.py \
  --data data/cleaned_environmental_data_10K.csv \
  --scaler checkpoints/environmental_baseline_scaler_10K.pkl \
  --checkpoint checkpoints/best_baseline_alien_protein_model_10K.pt \
  --loss-plot figures/train_loss_curve_baseline.png
\end{verbatim}
5) Evaluate both saved checkpoints on the held-out validation split and compute biological summaries:
\begin{verbatim}
python src/evaluation.py \
  --data data/cleaned_environmental_data_10K.csv \
  --scaler checkpoints/environmental_scaler_10K.pkl \
  --lstm-checkpoint checkpoints/best_alien_protein_model_10K.pt \
  --baseline-checkpoint checkpoints/best_baseline_alien_protein_model_10K.pt
\end{verbatim}
6) Recreate figures and data-visualization outputs:
\begin{verbatim}
python src/visualization_pca.py
python src/visualization_tsne.py
python src/exoplanet_data.py
python src/check_extreme_sites.py
\end{verbatim}
7) Launch the interactive Streamlit app for UI exploration:
\begin{verbatim}
streamlit run src/app.py
\end{verbatim}

Script descriptions:
\begin{itemize}
  \item `src/raw\_data.py': fetches UniProt sequences for GOLD taxonomy IDs, attaches GOLD
  coordinates, and queries NASA POWER / WorldClim / SoilGrids to build the
  raw merged Parquet dataset.
  \item `src/clean\_data.py': converts the Parquet to CSV, removes rows with numeric
  sentinel values (overflow/NoData markers), and writes the cleaned CSV used
  for training.
  \item `src/train.py': trains the conditional LSTM model and saves the best
  checkpoint, scaler, and a training loss curve.
  \item `src/train\_baseline.py': trains the simpler conditional RNN baseline and
  saves its own checkpoint and loss curve.
  \item `src/evaluation.py': loads checkpoints and computes statistical metrics
  (loss, perplexity, accuracy) and biological summaries (GRAVY) on the
  validation set.
  \item `src/visualization\_pca.py', `src/visualization\_tsne.py': generate the PCA
  and t-SNE sequence-space plots saved under `figures/`.
  \item `src/exoplanet\_data.py': fetches NASA Exoplanet Archive records and
  creates the exoplanet vs Earth environmental comparison figure.
  \item `src/check\_extreme\_sites.py': identifies and writes the listing of extreme
  sampling locations used in the manuscript (`results/extreme\_conditions.txt').

\end{itemize}

The annotations should be interpreted cautiously. The repeated type I
antifreeze hit is interesting because several generated sequences are
alanine-rich, matching a known feature of real type I antifreeze proteins, but

that do not imply a functional folded protein. The Cytochrome c class II hits

\section{Data and Model Limitations}
The main report summarizes the major limitations. Additional practical caveats
are listed here so the project is reproducible and interpretable:
\begin{itemize}
  \item GOLD coordinate coverage is sparse: only 64,665 of 530,160 local cached
  organism rows have both latitude and longitude.
  \item NASA POWER dates fall back to 2015-01-01 when GOLD collection dates are
  missing or unparseable, so some daily weather values are proxies rather than
  exact sampling-day measurements.
  \item WorldClim variables are long-term climatologies, not conditions measured
  at the exact sampling moment.
  \item The sparsity of extreme conditions in the training data means that
  model performance on those prompts should be interpreted with caution, especially
  for hot and hyper-humid conditions that are underrepresented or absent in the
  cleaned dataset.
  \item Multiple proteins from the same taxonomy ID can appear in both training
  and validation under the current random row split.
  \item Greedy decoding tends to produce low-complexity sequences. Future work
  should test temperature sampling, top-k/top-p sampling, repetition penalties,
  and larger training sets.
  \item Exoplanet plausibility depends on many variables absent from the model,
  including atmospheric pressure, atmospheric composition, liquid water,
  host-star spectrum, flare activity, ultraviolet exposure, gravity, and
  magnetic shielding.
\end{itemize}
